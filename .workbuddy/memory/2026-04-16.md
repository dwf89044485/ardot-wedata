- 用户决定将 wedata 的动效调参面板产品化为独立 npm 包
- 目标用户：Design Engineer
- 深度调研了竞品：leva（通用面板）、Tweakpane（纯 JS 面板）、Theatre.js（重量级动画编辑器）、VisBug（Google 的页面可视化设计调试工具）、agent-skills/tuning-panel（AI 生成调参面板）、Spring Visualizer 等
- 用户对 VisBug 的选区交互 + leva 的声明式 API 非常感兴趣，希望融合三者
- 输出了融合架构文档 `docs/motion-tuner-architecture.md`，核心设计：
  - Layer 1: 选区层（VisBug 式点击选中）

  - Layer 3: 工作流层（Motion Editor 的预览状态 + 代码导出）
  - 理想 API：一个 `useMotionTuner` hook 搞定一切
- 安装了 leva 让用户实际体验，接到了 Agent Card 真实参数上

## 架构方向确认（晚间讨论）

- 产品形态确定为：**纯 JS Core + 框架绑定层 + 配套 AI Skill**
- Core 层纯 JS，框架无关：面板 UI、选区引擎、状态机、参数 Store、导出引擎、快捷键、主题
- 框架绑定层是薄 wrapper：React hook / Vue composable / Vanilla 事件 API
- 预览状态管理是唯一需要框架绑定的功能
- 这个设计可以统一替代 leva + Tweakpane + lil-gui + dat.GUI 的碎片化格局
- 配套 AI Skill 负责自动参数发现和代码生成
